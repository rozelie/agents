## Source Code

### Inline Comments

Inlince comments need to always be prefixed with one of the following tags:

- `NOTE:`: Used to indicate noteworthy things that are not alarming
    - e.g. `# NOTE: Use X because Y requires 3x the amount of memory`
- `XXX:`: Used to indicate noteworthy things that are alarming
    - e.g. `# XXX: Using bad practice X because of Y`
- `TODO:`: Used to indicate work to be done in the future
    - e.g. `# TODO: Refactor this by doing X`
